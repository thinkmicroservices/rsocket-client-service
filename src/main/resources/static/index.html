RSocket Client Service

If you are seeing this page, you have successfully connected to the RSocketClient Service. This service exposes a set of REST endpoint methods that call corresponding methods on the RSocketService via RSocket connections. Both services can be run as either stand-alone jar files, or can be compiled into GraalVM native images and run from the included Docker-Compose file.

Stand-alone jar file execution

After building both the RSocketService and RSocketClientService (mvn clean package), start each application with the following command:

To start the RSocketService:

java -jar ./target/rsocket-service-0.0.1-SNAPSHOT.jar

And of course, to start the RSocketClientService:

java -jar ./target/rsocket-client-service-0.0.1-SNAPSHOT.jar

which, you have obviously already done if you are reading this.

With the application launched, you can exercise the service by clicking on each of the links below. Each link invokes the corresponding RSocket implementation mode.

  • Fire and Forget
  • Request/Response
  • Request/Stream
  • Channel

GraalVM native image and Docker-compose execution

If you want to run these services using Docker-Compose, you will need to build both using the following command:

mvn clean spring-boot:build-image
This command will build the GraalVM native image executable as well as generate the Docker container image. Once both container images have been prepared, they can both be started using the following command:

docker-compose It is important to remember that each service executes in its own container, with its own localhost. Therefore, when we run docker-compose, we override the RSocketClient's default RSocketServer host and port values. Additionally, we create a custom network (rsocket-example) to allow the services to communicate across.
